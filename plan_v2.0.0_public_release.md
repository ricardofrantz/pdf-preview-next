# v2.0.0 Plan — Public Release

Status: `[ ]` todo

## Prerequisite

`v1.10.0` (build and release automation) must be shipped and verified. Public
release work depends on a bundled, tested extension entrypoint and on the
guarded release workflow being in place.

## Goal

Make the extension ready for users beyond the local workflow: clear public docs,
current security posture, reproducible packages, and Marketplace/Open VSX
publishing.

## Scope

- Refresh README for public users.
- Add screenshots or a short GIF from the current UI.
- Document security model and PDF.js version.
- Document build, test, package, and install workflow.
- Verify Marketplace and Open VSX metadata.
- Publish only after explicit approval.

## Non-Goals

- No new viewer features.
- No telemetry.
- No broad API.
- No hidden dependency on another PDF extension.

## Likely Files

- `README.md`
- `CHANGELOG.md`
- `SECURITY.md`
- `CONTRIBUTING.md`
- `package.json`
- `.github/workflows/release.yml`
- `icon.png`
- release screenshots or media assets, if added

## Public Positioning

Use restrained claims:

- Lightweight PDF preview for VS Code.
- Modern PDF.js runtime.
- Reliable refresh for TeX/Typst/build workflows.
- Secure webview defaults.
- Practical controls: outline, find, refresh, print, source, keyboard navigation.

Avoid overclaiming:

- Do not claim complete PDF editor functionality.
- Do not claim fastest without benchmark evidence.
- Do not claim full parity with Firefox PDF.js viewer.

## Implementation Steps

1. Re-check version and changelog consistency.
2. Run a fresh package scan.
3. Install the VSIX locally with:
   - `code --install-extension pdf-preview-next-2.0.0.vsix --force`
4. Open representative PDFs:
   - normal PDF
   - outline PDF
   - password PDF
   - PDF regenerated by a build process
   - inter-PDF link fixture if `v1.7.0` landed
5. Capture screenshots/GIF from the current UI.
6. Update README:
   - install from VSIX
   - install from Marketplace/Open VSX once published
   - settings table
   - commands table
   - security model
   - build from source
   - known non-goals
7. Update SECURITY audit history if a new audit is performed.
8. Confirm package metadata:
   - display name
   - description
   - keywords
   - categories
   - icon
   - repository
   - bugs URL
9. Verify `engines.vscode` matches the lowest VS Code version actually
   tested, not the version bumped during modernization. If only newer VS
   Code releases have been tested, raise the floor rather than claiming
   compatibility that has not been verified.
10. Dry-run or guarded release workflow.
11. Ask for explicit approval before publishing.

## Tests

- `npm ci`
- `npm run format:check`
- `npm run lint`
- `npm run typecheck`
- `npm test`
- `npm audit --omit=dev --audit-level=high`
- `npm run package`
- `npm run package:scan -- pdf-preview-next-2.0.0.vsix`
- Install-and-open gate (mandatory):
  - `code --uninstall-extension ricardofrantz.pdf-preview-next`
  - `rm -rf ~/.vscode/extensions/ricardofrantz.pdf-preview-next-*`
  - Fully quit VS Code, then relaunch.
  - `code --install-extension pdf-preview-next-2.0.0.vsix --force`
  - Walk through the representative PDF set (normal, outline, password,
    rebuilt-by-build-process, inter-PDF link if `v1.7.0` shipped).
  - Confirm activation works on a *clean* VS Code install at the declared
    `engines.vscode` minimum, not just on the developer's local install.

## Acceptance Criteria

- README accurately describes current behavior.
- SECURITY accurately describes current security state.
- Built VSIX installs and opens PDFs locally.
- Marketplace/Open VSX publishing secrets are configured or the workflow clearly
  skips publishing.
- No plan files, tests, source maps, scratch directories, or runtime-local state
  are included in the release package.

## Risks

- Publishing is irreversible enough that version metadata should be checked
  twice.
- Marketplace search/discoverability depends heavily on name, description,
  keywords, icon, screenshots, and ratings. Technical quality alone will not
  make the extension visible.
